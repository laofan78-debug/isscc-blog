## 摘要

在ISSCC 2026的Processor Session中，韩国AI芯片独角兽**Rebellions**发表了题为《A Quad-Chiplet AI SoC with Full-Chip Scalable Mesh Over 16Gbps UCIe-Advanced Die-to-Die Interface for Large-Scale AI Inferencing》的论文，正式亮相其第二代AI加速器**REBEL-Quad**。这是**全球首款采用UCIe-Advanced互连标准的AI加速器**，也是韩国首颗突破光罩限制（Reticle Limit）的"Big Chip" AI半导体。

REBEL-Quad采用4个同构芯粒（Chiplet）通过UCIe-Advanced高速互连组成统一计算阵列，在三星4nm工艺下实现了**FP16 1PFLOPS、FP8 2PFLOPS**的算力，同时功耗控制在**600W**以内。在LLaMA 3.3 70B模型上实现了**56.8 TPS**的推理性能，能效表现优于NVIDIA H200。

本文将深入解析REBEL-Quad的架构设计、技术创新点及其对AI芯片发展的意义。

---

## 一、背景：为什么需要Chiplet架构？

### 1.1 光罩限制与良率困境

随着AI模型参数规模从数十亿向数千亿甚至万亿级演进，对计算芯片的面积和算力提出了前所未有的要求。然而，半导体制造中的**光罩限制（Reticle Limit）**成为了物理瓶颈——单片芯片的最大面积被光刻机的曝光范围所限制，通常约为800mm²。

当芯片面积接近光罩极限时，良率会急剧下降。根据半导体制造的经验公式，良率与芯片面积呈指数关系：

```
良率 = (1 - 缺陷密度)^芯片面积
```

这意味着一个800mm²的超大芯片，其良率可能只有30-40%，而将其拆分为4个200mm²的小芯片，每个小芯片的良率可以提升到85%以上，整体系统成本可降低30-50%。

### 1.2 Chiplet：突破物理极限的解决方案

Chiplet架构通过将大芯片拆分为多个小芯片（芯粒），使用先进封装技术重新集成，从根本上解决了上述问题：

| 对比维度 | 单片大芯片 | Chiplet架构 |
|---------|-----------|------------|
| 单片面积 | 接近光罩极限 (~800mm²) | 每片200-400mm² |
| 制造良率 | 30-40% | >85% |
| 制造成本 | 高 | 降低30-50% |
| 灵活性 | 固定配置 | 可扩展、模块化 |
| 互连带宽 | 片上总线 | 需要高速芯粒间互连 |

Rebellions敏锐地抓住了这一趋势，在REBEL-Quad上采用了**4芯粒同构设计**，每个芯粒约320mm²，通过**UCIe-Advanced**标准实现高速互连。

## 二、REBEL-Quad架构深度解析

### 2.1 整体架构概览

**关键规格：**
- **工艺**：Samsung 4nm (SF4X)
- **芯粒数量**：4个同构计算芯粒
- **互连标准**：UCIe-Advanced，16Gbps/lane
- **芯粒间带宽**：每通道1TB/s（双向）
- **芯粒间延迟**：仅11ns
- **内存**：4×HBM3E，总计144GB，4.8TB/s带宽
- **PCIe接口**：2×PCIe Gen5 x16
- **功耗**：最高600W

### 2.2 芯粒内部架构：粗粒度神经核心

每个Chiplet内部包含两个**Neural Core Cluster（神经核心集群）**，每个集群由8个**粗粒度神经核心（Coarse-Grained Neural Core）**组成，共享32MB片上存储。

#### 神经核心设计亮点

**（1）4MB大容量片上SRAM**

每个神经核心配备**4MB分布式SRAM**，这是REBEL-Quad最具特色的设计之一。相比传统NPU核心通常只有几十KB到几百KB的本地存储，4MB的容量允许：

- **更大的权重驻留**：减少HBM访问频率，降低功耗
- **细粒度数据复用**：在矩阵乘法中实现输入/权重的高复用率
- **更少的片外访存**：缓解内存墙问题

**（2）混合精度统一计算管线**

REBEL-Quad支持FP16和FP8混合精度计算，且**在单一计算管线中统一执行**，无需切换模式或重编译内核。这种设计的优势：

```
传统GPU设计：
  FP16计算块 ──┐
               ├──► 需要数据重排、内核重编译
  FP8计算块  ──┘

REBEL-Quad设计：
  统一混合精度管线 ──► 无缝切换，无需额外开销
```

根据Rebellions的数据，这种统一设计带来了**2.8倍的计算密度提升**（相比其第一代产品ATOM）。

**（3）预测性DMA引擎**

REBEL-Quad的DMA（直接内存访问）引擎采用**预测性预取策略**，与片上Mesh网络紧密耦合：

- 根据计算模式主动预取KV Cache数据
- 实现**2.7TB/s的有效带宽**（远超物理HBM带宽）
- 显著降低32K+长上下文LLM的Token级延迟

### 2.3 全芯片可扩展Mesh网络

REBEL-Quad最核心的技术创新是实现了**跨芯粒的全局Mesh网络**，将4个独立芯粒虚拟化为一个统一计算单元。

#### UCIe-Advanced：芯粒互连的新标准

UCIe（Universal Chiplet Interconnect Express）是行业统一的芯粒互连标准，而**UCIe-Advanced**是其高性能版本：

| 参数 | UCIe标准版 | UCIe-Advanced | 提升幅度 |
|-----|-----------|---------------|---------|
| 数据速率 | 2-8 Gbps | 最高16 Gbps | 2× |
| 每通道带宽 | 256-512 GB/s | 1 TB/s | 2-4× |
| 封装类型 | 标准封装 | 先进封装（CoWoS/EMIB） | - |
| 能效 | 较低 | 更高 | ~30% |

REBEL-Quad是**全球首款采用UCIe-Advanced的量产AI加速器**，每个芯粒通过UCIe-A接口与其他芯粒互联，形成**2×2的Mesh拓扑**。

#### Mesh网络的关键特性

**（1）256个路由器的硬件同步**

REBEL-Quad在全芯片范围内部署了**256个Mesh路由器**，实现硬件加速的全局同步。这意味着：

- 跨芯粒的数据同步无需软件干预
- 避免稀疏或不均衡负载下的流水线停顿
- 所有芯粒在分布式计算中保持高利用率

**（2）11ns超低延迟**

芯粒间通信延迟仅为**11ns**，这在很大程度上"隐藏"了芯粒边界，使得软件可以像操作单个大芯片一样编程REBEL-Quad，无需特别处理芯粒间通信。

**（3）透明扩展**

Rebellions强调REBEL-Quad的Mesh网络实现了**"Monolithic Efficiency with Modular Flexibility"**——既有模块化架构的灵活性，又能达到单片芯片的效率。

## 三、关键技术创新

### 3.1 集成硅电容器（Integrated Silicon Capacitor）

REBEL-Quad采用了**CoWoS-S（Chip-on-Wafer-on-Substrate with Silicon Interposer）**先进封装，并在每个芯粒下方集成了**硅电容器**。

**技术原理：**
- 硅电容器利用硅材料的深槽结构形成高电容密度
- 相比传统MLCC（多层陶瓷电容），硅电容可以实现：
  - **更低的ESR（等效串联电阻）**：减少电源纹波
  - **更高的频率响应**：有效抑制GHz级别的电源噪声
  - **更小的占位面积**：集成在封装内部，节省PCB空间

**效果：**
- 有效降低电源跌落（Power Droop），提升芯片稳定性
- 支持更高的工作频率和更低的电压
- 对HBM3E的高频访问提供更好的电源质量保障

### 3.2 双环电源管理机制

REBEL-Quad实现了精细的电源管理，支持多电压域动态调节：

- **计算域**：根据负载动态调整电压/频率
- **互连域**：独立管理UCIe链路的功耗
- **内存域**：优化HBM3E的功耗状态转换

### 3.3 软件栈：PyTorch原生支持

硬件只是AI加速器成功的一半，软件生态同样关键。REBEL-Quad提供完整的软件栈：

```
┌─────────────────────────────────────┐
│        PyTorch 2.x / vLLM          │
├─────────────────────────────────────┤
│      RBLN Compiler (图模式优化)      │
├─────────────────────────────────────┤
│      Triton Kernel (精度感知执行)    │
├─────────────────────────────────────┤
│      Runtime / Driver               │
├─────────────────────────────────────┤
│      REBEL-Quad Hardware            │
└─────────────────────────────────────┘
```

**关键特性：**
- **图模式优化**：自动算子融合、内存布局优化
- **精度感知执行**：自动选择最优精度（FP16/FP8）
- **零代码迁移**：标准PyTorch模型无需修改即可运行

## 六、技术启示与行业意义

### 6.1 Chiplet时代的AI芯片设计

REBEL-Quad的成功验证了Chiplet架构在AI加速器中的可行性：

1. **UCIe-Advanced是可行路径**：16Gbps的速率足以支撑AI计算的高带宽需求
2. **透明芯粒边界是关键**：11ns的延迟让软件几乎无感知
3. **先进封装是差异化要素**：CoWoS-S + 硅电容器的组合提供了竞争优势

### 6.2 韩国AI半导体的新里程碑

REBEL-Quad代表了韩国AI半导体产业的重要突破：

- **首次突破光罩限制**：韩国首颗"Big Chip" AI半导体
- **与全球巨头同台竞技**：ISSCC 2026上与AMD、IBM、MediaTek等一起发表
- **商业化能力验证**：从第一代ATOM到第二代REBEL-Quad，完成了从研发到量产的闭环

### 6.3 对行业的启示

**（1）开放生态 vs 封闭生态**

NVIDIA依靠NVLink构建封闭生态，而Rebellions选择拥抱**UCIe开放标准**。这可能预示着一个更开放的AI芯片生态正在形成。

**（2）能效优先于峰值算力**

REBEL-Quad的设计理念是**能效优先**——不追求最高的峰值算力，而是追求在实际工作负载中的高利用率和高能效。这与当前数据中心对可持续发展的要求高度契合。

**（3）软件生态的重要性**

Rebellions深知硬件只是基础，因此在软件栈上投入了大量资源，实现了与PyTorch/vLLM的无缝对接。这为其他AI芯片创业公司提供了借鉴。

Rebellions REBEL-Quad是ISSCC 2026上最具看点的AI芯片之一。它不仅展示了韩国半导体产业在AI芯片领域的最新进展，更重要的是验证了**UCIe-Advanced + Chiplet架构在量产AI加速器中的可行性**。

**核心亮点总结：**

1. ✅ **全球首款UCIe-Advanced AI加速器**：16Gbps/通道，1TB/s带宽，11ns延迟
2. ✅ **突破光罩限制**：4芯粒设计解决良率问题，同时保持单片芯片的使用体验
